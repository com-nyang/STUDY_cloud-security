## 궁금증 설명

### DMZ? 자세하게

`DMZ`는 `Demilitarized Zone`의 약자다. 원래 네트워크 구조에서는 내부망과 외부망 사이에 별도의 구역을 만들어, 외부에 공개할 장비를 그 구역에 두는 개념이다.

즉 원래 의미의 DMZ는:

- 내부 핵심 네트워크와
- 외부 인터넷 사이에
- 중간 공개 영역을 하나 두는 구조

라고 이해하면 된다.

하지만 일반 가정용 공유기에서 말하는 `DMZ`는 보통 그 구조를 완전히 구현하는 것이 아니라, `특정 내부 IP 하나를 외부에 매우 넓게 노출하는 기능`에 가깝다.

쉽게 말하면:

- 외부에서 공유기로 들어오는 요청 중
- 포트포워딩 등으로 따로 매핑되지 않은 요청을
- 특정 내부 장치 하나로 몰아주는 기능

으로 이해하면 된다.

그래서 가정용 공유기 메뉴의 DMZ는 "중간 구역을 만드는 것"이라기보다, "한 장치를 거의 외부에 직접 드러나게 하는 기능"처럼 동작하는 경우가 많다.

### 포트 포워딩과 차이

- `포트 포워딩`
  특정 포트만 골라서 내부 장치로 전달한다.
- `DMZ`
  훨씬 더 넓은 범위의 외부 요청을 특정 장치에 전달한다.

즉 포트포워딩은 선택적 전달이고, DMZ는 광범위한 전달에 가깝다.

### 일반 공유기에서 DMZ 기능이 왜 있는가? 어디에 쓰이는가?

이 기능이 있는 이유는, 어떤 장치나 서비스가 여러 포트를 쓰거나 어떤 포트를 써야 하는지 정확히 모를 때 일괄적으로 외부 연결을 열어보기 위한 용도로 쓰기 쉽기 때문이다.

대표적으로는:

- 게임기
- 테스트용 서버
- 특수한 네트워크 장비
- 특정 프로그램의 외부 연결 테스트

같은 환경에서 사용될 수 있다.

즉 사용자는 포트 하나하나 다 열지 않고, 일단 특정 장치를 외부 요청에 넓게 노출시켜 동작 여부를 확인할 수 있다.

### 왜 조심해서 봐야 하는가

DMZ는 편하지만 범위가 넓다.

포트포워딩은 "필요한 문만 연다"는 느낌이라면, DMZ는 "거의 대부분의 문을 한 장치 쪽으로 열어둔다"는 느낌에 가깝다.

그래서 이 메뉴는 단순 기능 설명 이상으로, `왜 포트포워딩과 별도 메뉴로 존재하는가`를 이해하는 게 중요하다. 즉 세밀한 선택 대신 넓은 노출을 제공하는 기능이기 때문이다.

## 한 번에 이해하기

가정용 공유기에서 `DMZ`는 보통 다음처럼 이해하면 된다.

- 특정 내부 장치 하나를 정한다.
- 외부에서 들어오는 다양한 요청을 그 장치 쪽으로 보낸다.
- 그래서 그 장치는 다른 내부 장치보다 훨씬 외부에 가깝게 노출된다.

즉 이 메뉴는 세밀한 포트 단위 제어가 아니라, 특정 장치를 넓게 외부 요청에 노출하는 기능이라고 보면 된다.
