## 궁금증 설명

### Spoofing

`Spoofing`은 자신을 다른 존재인 것처럼 속이는 행위를 뜻한다.

네트워크에서는 보통:

- IP 주소를 속이거나
- ARP 정보를 속이거나
- MAC 또는 장치 정보를 속이는 식

으로 나타난다.

예를 들어 `IP Spoof`는 공격자가 자기 IP가 아닌 다른 IP인 것처럼 패킷을 보내는 경우를 말한다. `ARP Spoof`는 같은 로컬 네트워크 안에서 장치들에게 잘못된 주소 정보를 주입하는 식이다.

즉 spoofing은 "출처를 속여서 통신을 교란하거나 우회하려는 행위"로 이해하면 된다.

### ICMP 패킷

`ICMP`는 `Internet Control Message Protocol`의 약자다. 인터넷 통신 상태를 알려주거나 진단할 때 사용하는 제어 메시지 프로토콜이다.

대표적으로:

- `ping`
  상대가 살아 있는지 확인
- 오류 메시지
  목적지 도달 불가 같은 상태 전달

같은 데 쓰인다.

즉 ICMP 패킷은 일반 웹 데이터 자체라기보다, 네트워크 상태 확인과 오류 전달을 위한 관리성 패킷이라고 보면 된다.

### Trace route

여기 적힌 `Trace rout`는 보통 `traceroute` 또는 `trace route`를 뜻한다.

이 기능은 내 패킷이 목적지까지 가는 동안 어떤 장비들을 거쳐 가는지 경로를 확인하는 도구다.

즉:

1. 패킷을 보낸다.
2. 중간 장비들이 응답한다.
3. 목적지까지 어떤 경로로 가는지 순서대로 본다.

그래서 `Trace route 응답 여부 설정`은 공유기가 이런 경로 추적 요청에 응답할지 말지를 정하는 항목으로 이해하면 된다.

### SNMP

`SNMP`는 `Simple Network Management Protocol`의 약자다. 네트워크 장비를 관리하거나 상태를 수집할 때 쓰는 프로토콜이다.

즉 공유기, 스위치, 서버 같은 장비의:

- 상태 정보
- 트래픽 정보
- 동작 정보

를 외부 관리 시스템이 가져갈 수 있게 해준다.

이 메뉴의 `SNMP 인입 허용 Rate`는 SNMP 관련 요청이 과도하게 들어오는 것을 제어하는 항목일 가능성이 높다.

## 항목을 읽는 방법

이 메뉴의 세부 항목은 보통 아래처럼 묶어서 이해하면 된다.

- `Flood / Scan 방어`
  너무 많은 요청이나 스캔성 접근을 막는 항목
- `Spoof 방어`
  주소를 속이는 공격을 막는 항목
- `Ping / traceroute 응답 설정`
  진단성 네트워크 요청에 얼마나 응답할지 정하는 항목
- `CPU 인입 이상 트래픽 제어`
  공유기 장비 자체에 과도한 트래픽이 몰리는 상황을 제어하는 항목
- `SNMP Rate`
  관리용 프로토콜 요청량을 제한하는 항목

## 한 번에 이해하기

`방화벽 > 보안 설정(DOS)`은 단순히 접속 허용/차단을 정하는 메뉴라기보다, 공유기가 비정상적으로 많은 요청이나 이상한 형태의 패킷을 받았을 때 어떻게 반응할지를 정하는 페이지다.

즉 이 메뉴는 "정상 통신을 여는 기능"보다 "비정상 트래픽을 견디고 걸러내는 기능"에 더 가깝다고 이해하면 된다.
