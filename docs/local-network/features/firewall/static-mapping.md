# 방화벽 > STATIC MAPPING

## 위치

`방화벽 > STATIC MAPPING`

## 기능 설명

출발지 IP, Port를 보고 IP, Port로 연결시켜주는 설정을 할 수 있는 페이지입니다.

## 보통 여기서 보는 항목

- Static Mapping 설정
- Static Mapping Table



## 궁금한거

- Static Mapping?

## 궁금증 설명

### Static Mapping

`Static Mapping`은 말 그대로 `고정된 매핑 규칙`을 만든다는 뜻이다.

여기서 `mapping`은 한쪽의 주소/포트를 다른 쪽 주소/포트에 대응시키는 것을 말한다. 지금 적어둔 설명대로라면, 이 메뉴는 단순히 IP를 고정하는 기능이 아니라 `특정 출발지 IP/포트 조건에 따라 특정 목적지 IP/포트로 연결되도록 정하는 기능`에 가깝다.

쉽게 말하면:

- 어떤 요청이 들어왔을 때
- 그 요청의 주소나 포트 조건을 보고
- 미리 정해둔 다른 주소/포트로 연결시킨다

이런 `고정 규칙 테이블`을 만드는 메뉴라고 이해하면 된다.

### 왜 Static이라고 부르나

`Static`이라는 말이 붙는 이유는 이 규칙이 자동으로 바뀌지 않고, 사용자가 직접 정한 값으로 고정되어 있기 때문이다.

즉:

- 자동 판단이 아니라
- 사용자가 수동으로
- "이 조건이면 저쪽으로 보낸다"를 정해놓는 방식

이다.

### 포트 포워딩과는 뭐가 다른가

이름이 비슷해 보이지만, 보통은 아래처럼 이해하면 된다.

- `포트 포워딩`
  외부에서 들어온 특정 포트를 내부 특정 장치의 포트로 보낸다
- `Static Mapping`
  더 일반적인 주소/포트 매핑 규칙 테이블일 수 있다

즉 포트 포워딩이 "특정 포트를 어디로 보낼지"에 집중한 메뉴라면, Static Mapping은 주소와 포트를 묶어서 더 고정된 규칙 형태로 다루는 메뉴일 가능성이 있다.

### 이 메뉴를 볼 때 확인할 점

실제 화면에서는 아래를 보면 이 기능 성격을 더 정확히 알 수 있다.

- 입력 항목에 `출발지 IP`가 있는지
- `목적지 IP`와 `목적지 포트`를 따로 넣는지
- 프로토콜(TCP/UDP)을 고를 수 있는지
- 내부 장치 고정 주소용 메뉴인지, 트래픽 변환용 메뉴인지

즉 이 메뉴는 이름만으로 단정하기보다, `무엇을 입력하게 되어 있는가`를 보고 판단하는 게 중요하다.

## 한 번에 이해하기

`방화벽 > STATIC MAPPING`은 고정된 네트워크 매핑 규칙을 만드는 메뉴로 이해하면 된다.

- 어떤 주소/포트 조건이 들어오면
- 어디로 보낼지
- 어떤 방식으로 연결시킬지

를 수동으로 정해두는 성격의 기능이다.

따라서 이 메뉴는 단순 상태 확인 메뉴가 아니라, 트래픽이 어떤 경로로 흘러가게 할지를 고정 규칙으로 설계하는 페이지라고 볼 수 있다.
