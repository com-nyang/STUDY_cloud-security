## 궁금증 설명

### DDNS

`DDNS`는 `Dynamic DNS`의 약자다. 바뀔 수 있는 IP 주소를 고정된 이름과 연결해주는 기능이다.

보통 집 인터넷 환경에서는 공인 IP가 계속 고정되어 있지 않고 바뀔 수 있다. 그런데 사람이 숫자 IP를 매번 확인해서 쓰기는 불편하다.

그래서 DDNS는:

1. 현재 공인 IP가 무엇인지 확인하고
2. 그 IP를 특정 도메인 이름에 연결해두고
3. IP가 바뀌면 그 연결 정보를 다시 갱신한다

는 식으로 동작한다.

쉽게 말하면:

- 숫자로 된 주소는 바뀔 수 있지만
- 사람이 기억하는 이름은 유지되게 해주는 기능

이다.

예를 들어:

- 공인 IP는 오늘 `1.2.3.4`
- 내일 `5.6.7.8`

로 바뀔 수 있어도,

- `myhome.exampleddns.com`

같은 이름은 계속 그대로 쓸 수 있게 만드는 것이다.

### 왜 공유기에 이 기능이 있는가

공유기는 인터넷과 직접 연결되는 장비이기 때문에 현재 외부 IP가 무엇인지 가장 먼저 알고 있는 장비다.

그래서 공유기가 DDNS 서비스와 직접 연동하면:

- 공인 IP가 바뀔 때
- 공유기가 자동으로 DDNS 서버에 알리고
- 도메인 정보를 최신 상태로 유지할 수 있다.

즉 공유기 메뉴에 DDNS가 있다는 것은, 공유기 자체가 "현재 외부 주소 변경"을 감지하고 이름 정보를 갱신하는 역할을 할 수 있다는 뜻이다.

## 한 번에 이해하기

`관리 > DDNS`는 숫자로 바뀌는 외부 IP 주소를, 사람이 기억하기 쉬운 고정 이름과 연결하는 메뉴다.

- `서비스 공급자`
  어떤 DDNS 서비스를 쓸지 정한다.
- `도메인 이름`
  사용할 고정 이름을 입력한다.
- `사용자 이름/이메일`, `비밀번호/키`
  해당 DDNS 서비스 계정 정보를 입력한다.

즉 이 메뉴는 외부 주소가 바뀌더라도 일정한 이름으로 공유기를 찾을 수 있게 만드는 기능이라고 이해하면 된다.
