# 관리 > 원격 전송 설정

## 위치

`관리 > 원격 전송 설정`

## 기능 설명

원격 전송 레벨을 설정하기 위한 페이지입니다.

## 보통 여기서 보는 항목

- Critical
- Warn
- Info
- Debug

## 궁금한거

- 원격 전송 레벨?

## 궁금증 설명

### 원격 전송 레벨

`원격 전송 레벨`은 공유기에서 발생한 로그나 이벤트를 외부로 보낼 때, 어느 심각도 수준까지 보낼지를 정하는 기준으로 볼 수 있다.

즉 공유기 안에서 일어난 일들을:

- 전부 다 보낼지
- 중요한 것만 보낼지
- 아주 심각한 문제만 보낼지

를 정하는 설정이다.

이 메뉴에 `Critical`, `Warn`, `Info`, `Debug`가 보인다면, 보통 로그 심각도 레벨을 뜻한다.

- `Critical`
  매우 심각한 오류나 장애 수준
- `Warn`
  경고성 이벤트
- `Info`
  일반적인 동작 정보
- `Debug`
  가장 자세한 디버깅용 정보

### 왜 레벨을 나누는가

로그를 너무 많이 보내면 관리가 어려워지고, 너무 적게 보내면 필요한 정보를 놓칠 수 있다.

그래서 장비는 보통 중요도에 따라 로그를 나눈다.

- 중요한 장애만 보고 싶으면 `Critical`
- 경고와 이상 징후까지 보고 싶으면 `Warn`
- 일반 동작 흐름도 보고 싶으면 `Info`
- 아주 세세한 내부 동작까지 보고 싶으면 `Debug`

라는 식으로 생각하면 된다.

### 원격 전송이란 무엇인가

여기서 `원격 전송`은 공유기 안에만 로그를 두지 않고, 외부의 다른 장비나 서버로 보내는 것을 뜻할 가능성이 높다.

즉:

- 공유기 내부에서만 보는 로그가 아니라
- 다른 관리 시스템이나 로그 서버로
- 이벤트를 전달하는 기능

으로 이해하면 된다.

## 한 번에 이해하기

`관리 > 원격 전송 설정`은 "공유기 로그를 외부로 보낼 때 얼마나 자세한 수준까지 보낼 것인가"를 정하는 메뉴라고 보면 된다.

즉 이 메뉴는 네트워크 연결 자체를 조정하는 기능이 아니라, 공유기 운영 기록을 외부에서 어떻게 수집하고 관리할지를 정하는 운영용 설정 페이지다.
