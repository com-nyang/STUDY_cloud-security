# 관리 > 트래픽 통계

## 위치

`관리 > 트래픽 통계`

## 기능 설명

유무선 네트워크에 관한 데이터 송수신량을 보여주는 페이지입니다.

## 보통 여기서 보는 항목

- 무선 랜 5G
- 무선 랜 2.4G
- 로컬 랜
- 인터넷
- CRC 정보

## 궁금한거

- CRC?

## 궁금증 설명

### CRC

`CRC`는 `Cyclic Redundancy Check`의 약자다. 데이터를 전송할 때 내용이 중간에 깨졌는지 확인하기 위한 오류 검사용 값이다.

쉽게 말하면:

1. 데이터를 보낼 때
2. 그 데이터가 정상인지 확인할 수 있는 검사용 값을 함께 붙여 보낸다.
3. 받는 쪽에서 다시 계산해보고
4. 값이 다르면 "전송 중 뭔가 문제가 있었구나"라고 판단한다.

즉 CRC는 데이터가 손상되었는지 확인하는 검증 장치라고 보면 된다.

### 트래픽 통계에 왜 CRC가 보이는가

트래픽 통계는 단순히 "얼마나 많이 보냈는가"만 보는 화면이 아니라, "전송 품질이 괜찮은가"까지 같이 보려는 경우가 많다.

그래서 CRC 정보가 보인다면 보통은:

- 전송 오류가 얼마나 있었는지
- 링크 품질에 문제가 있는지
- 케이블이나 무선 환경이 불안정한지

같은 점을 간접적으로 판단하는 데 쓰일 수 있다.

### CRC 값이 의미하는 것

- CRC 오류가 거의 없으면:
  전송 품질이 비교적 정상이라고 볼 수 있다.
- CRC 오류가 많으면:
  데이터가 전송 중 깨지는 일이 자주 생긴다는 뜻일 수 있다.

이 경우에는:

- 케이블 문제
- 포트 문제
- 무선 간섭
- 장비 상태 문제

등을 의심해볼 수 있다.

## 한 번에 이해하기

`관리 > 트래픽 통계`는 단순 사용량 화면이 아니다.

- `무선 랜 5G / 2.4G`
  무선 구간의 송수신량
- `로컬 랜`
  내부 유선 네트워크 트래픽
- `인터넷`
  외부 인터넷 방향 트래픽
- `CRC 정보`
  전송 품질 이상이 있었는지 보여주는 참고 정보

즉 이 메뉴는 `얼마나 많이 통신했는가`와 `그 통신 품질이 괜찮았는가`를 함께 보는 통계 화면이라고 이해하면 된다.
