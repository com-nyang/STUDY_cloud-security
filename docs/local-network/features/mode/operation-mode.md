# 운용모드 설정

## 위치

`운용모드 설정`

## 기능 설명

네트워크 모드를 변경할 수 있는 페이지입니다.

## 보통 여기서 보는 항목

- Bridge Mode
- Gateway Mode
- Repeater Mode

## 이해 포인트

운용모드는 공유기가 네트워크에서 어떤 역할을 맡는지 정하는 설정이다.  
같은 장비라도 모드에 따라 NAT, DHCP, 방화벽, 무선 확장 방식이 달라진다.

## 모드별 설명

### Bridge Mode

상위 장비가 이미 라우팅을 하고 있을 때, 이 장비를 중간 연결 장치처럼 동작시키는 모드다.

- 보통 NAT 기능을 직접 하지 않거나 최소화한다
- DHCP 서버 기능도 꺼지거나 제한되는 경우가 많다
- 기존 메인 공유기 또는 상위 라우터 뒤에서 네트워크를 단순 확장할 때 사용한다

보안 관점:

- 방화벽과 주소 분리가 약해질 수 있으므로 실제 보호 경계가 어디인지 다시 확인해야 한다
- 관리자 페이지 접근 경로가 상위 네트워크와 어떻게 연결되는지 확인해야 한다

### Gateway Mode

공유기가 인터넷과 내부 네트워크 사이의 기본 관문 역할을 하는 일반적인 모드다.

- NAT 수행
- DHCP 서버 제공
- 내부망과 외부망 사이의 기본 방화벽 역할 수행
- 가정이나 소규모 사무실에서 가장 흔하게 쓰는 기본 모드다

보안 관점:

- 포트 포워딩, DMZ, 원격 관리, UPnP 같은 외부 노출 기능을 특히 주의해서 봐야 한다
- 내부 장치가 모두 이 장비를 경유하므로 로그와 트래픽 관리 지점으로도 중요하다

### Repeater Mode

기존 무선 신호를 받아서 다시 중계해 무선 범위를 넓히는 모드다.

- 유선 연결 없이 무선 커버리지를 확장할 때 사용한다
- 메인 AP와 중계 장비 사이의 무선 품질에 성능이 크게 좌우된다
- 환경에 따라 속도 저하나 지연 증가가 생길 수 있다

보안 관점:

- 어떤 SSID와 보안 설정을 그대로 중계하는지 확인해야 한다
- 약한 암호화 방식이나 오래된 무선 설정을 그대로 확장하면 위험도 함께 늘어난다
- 관리 페이지가 어느 쪽 네트워크에서 접근 가능한지 확인해야 한다

## 관련 기능과의 연결

- Gateway Mode는 포트 포워딩, DMZ, IP 필터링, DDNS와 같이 봐야 한다
- Bridge Mode는 로컬 랜 설정과 상위 장비의 DHCP/방화벽 구조를 함께 봐야 한다
- Repeater Mode는 무선 기본 설정, 보안 설정, AP 검색 기능과 연결해서 봐야 한다
