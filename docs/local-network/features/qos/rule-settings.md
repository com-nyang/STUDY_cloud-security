## 궁금증 설명

### QOS

`QoS`는 `Quality of Service`의 약자다. 네트워크 트래픽의 우선순위를 조정해서 중요한 통신이 더 안정적으로 처리되도록 하는 개념이다.

쉽게 말하면:

- 네트워크를 사용하는 데이터가 많을 때
- 모든 트래픽을 똑같이 취급하지 않고
- 어떤 것은 더 우선해서 보내고
- 어떤 것은 나중에 처리하게 만드는 기능

이다.

예를 들어:

- 영상 통화
- 게임
- 스트리밍
- 일반 다운로드

가 동시에 일어날 때, QoS는 지연에 민감한 트래픽을 더 우선하게 만들 수 있다.

### ToS

`ToS`는 `Type of Service`의 약자다. IP 패킷에 "이 트래픽은 어떤 성격인가"를 표시하던 개념이다.

쉽게 말하면 패킷에 붙는 서비스 성격 정보라고 볼 수 있다.

이 개념은 예전부터 쓰였고, 오늘날에는 더 세분화된 `DSCP` 개념으로 이어지는 흐름으로 이해하면 된다.

### DSCP

`DSCP`는 `Differentiated Services Code Point`의 약자다.

이것도 패킷에 붙는 우선순위/서비스 구분 정보인데, QoS에서 어떤 트래픽을 더 우선할지 판단할 때 참고할 수 있다.

즉:

- `ToS`
  예전부터 쓰인 서비스 구분 개념
- `DSCP`
  그와 관련된 현대적 우선순위 표시 방식

으로 이해하면 된다.

공유기에서 `ToS/DSCP` 항목이 보이면, 해당 규칙이 특정 우선순위 표시를 가진 트래픽에 적용되도록 설정할 수 있다는 뜻일 가능성이 높다.

## 항목을 읽는 방법

이 페이지는 보통 "어떤 트래픽을 찾아서 어떻게 처리할 것인가"를 규칙으로 만드는 구조다.

- `IPMode`
  IPv4 또는 IPv6 같은 주소 체계 기준
- `포트`
  서비스 포트 번호 조건
- `Rule 종류`
  규칙 유형
- `Physical 포트`
  실제 LAN/WAN 포트 기준 조건
- `VLAN`
  특정 VLAN 트래픽 기준 조건
- `프로토콜`
  TCP, UDP 같은 통신 종류
- `ToS/DSCP`
  우선순위 표시 기준
- `출발지`
  어디서 오는 트래픽인지
- `목적지`
  어디로 가는 트래픽인지
- `Action`
  우선순위를 높일지, 제한할지, 어떤 방식으로 처리할지

즉 QoS 규칙은 "이 조건에 맞는 트래픽은 이렇게 다뤄라"라고 적는 정책표에 가깝다.

## 한 번에 이해하기

`QOS > RULE 설정`은 네트워크 속도를 직접 올리는 마법 기능이 아니라, 트래픽이 몰릴 때 어떤 통신을 더 우선하고 어떤 통신을 덜 우선할지를 정하는 메뉴다.

즉 이 메뉴는 대역폭을 새로 만드는 기능이 아니라, `기존 대역폭을 어떤 기준으로 나눠 쓸 것인가`를 정하는 규칙 설정 페이지라고 이해하면 된다.
