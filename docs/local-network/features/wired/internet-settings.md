## 궁금증 설명

### 그래서 여긴 뭐 하는 페이지인가

이 페이지는 공유기가 `외부 인터넷과 어떻게 연결될지`를 정하는 페이지다.

내부 네트워크를 관리하는 메뉴가 아니라, 공유기의 `WAN 쪽 설정`을 다루는 곳이라고 보면 된다. 즉, 공유기가 통신사 회선으로부터 어떤 방식으로 IP를 받고, 어떤 DNS를 쓰고, 어떤 부가 네트워크 기능을 켤지 정하는 메뉴다.

쉽게 말하면:

- `로컬 랜 설정`
  공유기 안쪽 네트워크를 다룬다.
- `인터넷 설정`
  공유기 바깥쪽 인터넷 연결을 다룬다.

그래서 이 페이지를 이해하면 "공유기는 외부 인터넷에 어떻게 붙어 있는가"를 볼 수 있다.

### MTU

`MTU`는 `Maximum Transmission Unit`의 약자다. 한 번에 보낼 수 있는 데이터 패킷의 최대 크기를 뜻한다.

쉽게 말하면 네트워크가 한 번에 운반하는 데이터 조각의 크기다.

- 너무 크면:
  중간 장비가 그 크기를 처리하지 못해서 잘리거나 문제가 생길 수 있다.
- 너무 작으면:
  데이터를 더 잘게 나눠 보내야 해서 비효율적일 수 있다.

대부분 기본값으로 충분하지만, 특정 회선이나 VPN 환경에서는 MTU 값이 연결 품질에 영향을 줄 수 있다.

### IGMP 프록시

`IGMP 프록시`는 멀티캐스트 트래픽을 다른 네트워크 구간으로 전달해주는 기능이다.

대표적으로 IPTV 같은 서비스에서 자주 등장한다. 여러 장치가 같은 방송 스트림을 받는 환경에서, 공유기가 그 멀티캐스트 트래픽을 적절히 전달하도록 돕는 역할을 한다.

즉:

- `IGMP`
  누가 어떤 멀티캐스트 그룹에 참여하는지 관리하는 프로토콜
- `IGMP 프록시`
  그 멀티캐스트 요청과 트래픽을 네트워크 사이에서 중계해주는 기능

일반 웹브라우징보다 IPTV, 셋톱박스, 멀티캐스트 서비스 쪽에서 더 중요한 항목이다.

### IPsec

`IPsec`은 `Internet Protocol Security`의 약자다. IP 계층에서 통신을 보호하기 위한 보안 프로토콜 묶음이다.

주로 VPN에서 많이 사용된다. 즉, 인터넷 위에 안전한 터널을 만들어서 데이터를 보호하는 방식이다.

이 메뉴의 `IPsec pass through`는:

- 공유기 뒤에 있는 장치가
- IPsec 기반 VPN에 접속하려고 할 때
- 그 트래픽이 공유기를 통과할 수 있게 도와주는 기능

즉 공유기 자체가 VPN 서버라는 뜻이 아니라, 내부 장치의 VPN 통신을 막지 않고 통과시키는 옵션으로 이해하면 된다.

### PPTP

`PPTP`는 `Point-to-Point Tunneling Protocol`의 약자다. 예전부터 쓰이던 VPN 방식 중 하나다.

현재는 오래된 방식으로 보는 편이고, 현대 기준에서는 상대적으로 덜 선호된다. 다만 일부 오래된 장비나 환경에서는 아직 남아 있을 수 있다.

`PPTP pass through`는 내부 장치가 PPTP 기반 VPN에 접속할 때 그 연결이 공유기를 통과할 수 있도록 해주는 기능이다.

### L2TP

`L2TP`는 `Layer 2 Tunneling Protocol`의 약자다. 이것도 VPN과 관련된 터널링 프로토콜이다.

L2TP 자체는 암호화를 직접 담당하지 않는 경우가 많아서, 실제로는 다른 보안 방식과 함께 사용되기도 한다.

이 메뉴의 `L2TP pass through`는 내부 장치가 L2TP 기반 VPN 연결을 할 때 그 트래픽을 공유기가 통과시키도록 도와주는 기능이다.

## 한 번에 이해하기

`유선설정 > 인터넷 설정`은 공유기의 `외부 연결 성격`을 정하는 메뉴다.

- `IP 획득 방법`
  공인 IP를 어떤 방식으로 받을지 정한다.
- `DNS`
  도메인 이름을 실제 IP로 바꿔줄 서버를 정한다.
- `MTU`
  네트워크 데이터 조각 크기와 관련된다.
- `MAC Clone`
  특정 장치의 MAC 주소를 흉내 내야 할 때 쓰일 수 있다.
- `IGMP 프록시`
  IPTV 같은 멀티캐스트 트래픽과 관련된다.
- `IPsec / PPTP / L2TP pass through`
  내부 장치의 VPN 연결이 공유기를 통과할 수 있게 도와준다.

즉 이 메뉴는 단순히 인터넷 on/off를 정하는 곳이 아니라, 공유기가 외부 인터넷과 어떤 방식으로 연결되고 어떤 보조 네트워크 기능을 허용할지를 정하는 페이지라고 이해하면 된다.

## 추가 설명

### Tunneling

`Tunneling`은 말 그대로 "터널을 만든다"는 개념이다. 네트워크에서는 어떤 데이터를 다른 데이터 안에 넣어서 보내는 방식을 뜻한다.

쉽게 말하면:

1. 원래 보내고 싶은 데이터가 있다.
2. 그 데이터를 바로 보내지 않고
3. 다른 규칙의 포장 안에 넣어서 보낸다.

이걸 왜 하냐면, 원래는 바로 지나가기 어렵거나 특별한 방식으로 보내야 하는 데이터를 네트워크를 통해 전달하기 위해서다.

예를 들어 VPN을 생각하면 이해하기 쉽다.

- 내 장치와 회사 네트워크 사이에
- 인터넷 위로
- 가상의 전용 통로처럼 보이게 연결을 만든다.

이때 실제로는 그냥 인터넷을 쓰지만, 데이터는 `터널 안을 지나가는 것처럼` 포장되어 이동한다. 그래서 `tunneling`이라고 부른다.

### VPN과 Tunneling의 관계

- `VPN`
  가상 사설망이라는 전체 개념에 가깝다.
- `Tunneling`
  그 가상 연결을 만들기 위해 데이터를 감싸서 전달하는 방식이다.

즉, VPN은 목적이고 tunneling은 그 목적을 구현하는 방법 중 하나라고 보면 된다.

### PPTP, L2TP, IPsec과의 관계

- `PPTP`
  터널링 방식의 한 종류
- `L2TP`
  터널링 방식의 한 종류
- `IPsec`
  보안 보호 기능을 제공하는 방식이며, VPN 구성에서 함께 쓰이거나 단독으로 쓰일 수 있다

그래서 이 메뉴에 PPTP, L2TP, IPsec pass through가 있는 것은 "내부 장치가 이런 VPN/터널링 기반 연결을 사용할 때 공유기가 그 통신을 지나가게 해줄 것인가"를 다루는 것이라고 이해하면 된다.
