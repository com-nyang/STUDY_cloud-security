# 유선설정 > IPv6 설정

## 위치

`유선설정 > IPv6 설정`

## 기능 설명

IPv6 주소 설정을 위한 페이지입니다.

## 보통 여기서 보는 항목

- IPv6 설정
  
  - IPv6 방식 선택
  
  - IPv6 주소
  
  - Prefix 길이
  
  - 기본 게이트웨이
  
  - 자동으로 DNS 서버 주소 받기
  
  - 다음 DNS 서버 주소 사용

- IPv6 PassThrough 설정

## 여기서 드는 궁금증

- Prefix?

- 인터넷 설정이랑 왜 분리되어 있는지

## 궁금증 설명

### Prefix

`Prefix`는 IPv6 주소에서 `네트워크 부분이 어디까지인가`를 나타내는 개념이다.

IPv4에서 서브넷 마스크가 네트워크 범위를 나누는 역할을 했다면, IPv6에서는 보통 `Prefix 길이`가 그 역할을 한다고 보면 된다.

예를 들면:

- IPv4에서는
  `192.168.0.10 / 255.255.255.0`
- IPv6에서는
  `2001:db8::1234/64`

여기서 `/64`가 prefix 길이다. 즉 "앞의 64비트는 네트워크를 나타내고, 나머지는 장치 주소에 가깝다"는 뜻이다.

쉽게 말하면 prefix는 "이 주소 중 어디까지가 같은 네트워크인가"를 알려주는 기준이다.

### 인터넷 설정이랑 왜 분리되어 있는지

이유는 IPv4와 IPv6가 서로 다른 주소 체계이기 때문이다.

- `인터넷 설정`
  보통 IPv4 기준 WAN 연결을 다룬다.
- `IPv6 설정`
  IPv6 기준 주소, 게이트웨이, DNS, 전달 방식을 따로 다룬다.

둘 다 인터넷 연결과 관련은 있지만, 주소 형식과 동작 방식이 다르기 때문에 메뉴를 나눠두는 경우가 많다.

쉽게 말하면:

- IPv4는 기존에 많이 쓰던 인터넷 주소 체계
- IPv6는 주소 공간이 훨씬 큰 새로운 주소 체계

공유기 입장에서는 둘을 동시에 지원할 수도 있고, 한쪽만 쓸 수도 있기 때문에 별도 설정 메뉴로 분리해두는 것이다.

## 한 번에 이해하기

`유선설정 > IPv6 설정`은 "이 공유기가 IPv6 인터넷을 어떻게 쓸 것인가"를 정하는 메뉴다.

- `IPv6 방식 선택`
  IPv6를 어떤 방식으로 사용할지 정한다.
- `IPv6 주소`
  공유기가 받는 IPv6 주소를 보여준다.
- `Prefix 길이`
  이 IPv6 주소가 어떤 네트워크 범위에 속하는지 보여준다.
- `기본 게이트웨이`
  IPv6 기준 외부로 나갈 때 거치는 경로를 뜻한다.
- `DNS 서버`
  IPv6 환경에서 이름 해석에 사용할 서버를 뜻한다.
- `IPv6 PassThrough`
  IPv6 트래픽을 어떤 방식으로 통과시킬지와 관련된 설정일 수 있다.

즉 이 메뉴는 단순히 "주소 하나 더 있는 메뉴"가 아니라, IPv4와는 별도의 인터넷 주소 체계를 따로 다루는 페이지라고 이해하면 된다.
