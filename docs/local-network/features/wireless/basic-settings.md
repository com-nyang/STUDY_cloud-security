## 궁금증 설명

### Control Sideband

`Control Sideband`는 넓은 채널 폭을 사용할 때, 기준이 되는 주 채널이 어느 쪽에 놓이는지를 정하는 개념이다.

조금 쉽게 말하면:

- 무선 채널을 넓게 쓰면
- 하나의 채널만 쓰는 게 아니라
- 인접한 영역까지 묶어서 쓰게 된다.

이때 기준 채널이 아래쪽인지 위쪽인지 정하는 것이 `Control Sideband`다.

예를 들어 40MHz 채널 폭을 쓸 때:

- `lower`
  기준 채널이 아래쪽
- `upper`
  기준 채널이 위쪽

일반 사용자는 이 값을 자주 직접 조정하지 않아도 되지만, 무선 채널 구조를 세밀하게 맞출 때 등장하는 항목이다.

### TX

`TX`는 `Transmit`의 약자다. 데이터를 `보내는 쪽`을 뜻한다.

무선 설정에서 `TX 제한`이 있다면, 보통 공유기가 무선으로 데이터를 보내는 양이나 속도, 혹은 출력 관련 동작을 제한하는 의미로 볼 수 있다.

쉽게 말하면:

- `TX`
  송신
- 공유기에서 바깥으로 나가는 무선 데이터

### RX

`RX`는 `Receive`의 약자다. 데이터를 `받는 쪽`을 뜻한다.

`RX 제한`은 보통 공유기가 무선으로 받는 데이터 양이나 처리 범위를 제한하는 의미로 볼 수 있다.

즉:

- `RX`
  수신
- 바깥 장치에서 공유기로 들어오는 무선 데이터

그래서 `TX / RX`는 네트워크에서 아주 기본적으로 나오는 표현이고, 각각 `송신 / 수신`으로 이해하면 된다.

### Multi SSID

`Multi SSID`는 하나의 공유기에서 Wi-Fi 이름을 여러 개 운영하는 기능이다.

예를 들어 한 공유기에서:

- `HomeWiFi`
- `GuestWiFi`
- `IoTWiFi`

같이 여러 SSID를 동시에 만들 수 있다.

왜 쓰냐면 용도를 나누기 위해서다.

- 가족용 네트워크
- 손님용 네트워크
- IoT 전용 네트워크

이런 식으로 구분하면 같은 공유기를 쓰더라도 무선 네트워크를 논리적으로 나눌 수 있다.

## 한 번에 이해하기

`무선 설정 > 기본 설정`은 "이 공유기의 Wi-Fi를 어떤 모습으로 운영할 것인가"를 정하는 메뉴다.

- `Band`
  2.4GHz / 5GHz 같은 주파수 대역
- `SSID`
  사용자가 보는 Wi-Fi 이름
- `채널 폭`
  무선 채널을 얼마나 넓게 쓸지
- `채널 번호`
  실제로 어떤 채널을 사용할지
- `SSID 알림`
  Wi-Fi 이름을 주변 기기에 보이게 할지
- `데이터 전송률`
  무선 속도 관련 기본 설정
- `TX / RX 제한`
  송신/수신 동작 제한
- `Multi SSID`
  Wi-Fi 이름을 여러 개 운영하는 기능
- `동시접속제한`
  접속 가능한 장치 수 제한

즉 이 메뉴는 무선 네트워크의 `이름`, `대역`, `채널`, `송수신 동작`, `접속 구조`를 정하는 가장 기본적인 페이지라고 이해하면 된다.
