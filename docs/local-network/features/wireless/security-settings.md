# 무선 설정 > 보안 설정

## 위치

`무선 설정 > 보안 설정`

## 기능 설명

무선 인터넷의 보안을 설정하는 페이지입니다.

WEP 또는 WPA 암호키를 사용하여 권한없이 무선 인터넷에 접근하는것을 예방할 수 있습니다.

## ## 보통 여기서 보는 항목

- 암호하
- 인증모드 (RADIUS, Pre-Shared Key)
- WPA Cipher Suite
- WPA2 Cipher Suite
- Key 형식
- Pre-Shared Key

## 여기서 궁금한거

- WPA?

- WEP?

- RADIUS?

- Pre-Shared Key?

## 궁금증 설명

### WPA

`WPA`는 `Wi-Fi Protected Access`의 약자다. 무선 네트워크를 보호하기 위한 보안 방식이다.

쉽게 말하면 Wi-Fi 접속 시 인증과 암호화를 어떻게 할지를 정하는 표준이다.

보통은 다음 흐름으로 발전했다.

- `WEP`
  오래된 방식
- `WPA`
  그보다 개선된 방식
- `WPA2`
  훨씬 널리 사용된 방식
- `WPA3`
  더 최신 방식

즉 `WPA`는 무선 네트워크 보안 방식의 한 종류라고 보면 된다.

### WEP

`WEP`는 `Wired Equivalent Privacy`의 약자다. 매우 오래된 Wi-Fi 보안 방식이다.

이름만 보면 안전해 보이지만, 현재 기준으로는 취약한 방식으로 알려져 있다. 그래서 지금은 일반적으로 권장되지 않는다.

쉽게 말하면:

- 예전 Wi-Fi 보안 방식
- 현재는 보안성이 부족하다고 보는 편

그래서 무선 보안 설정을 볼 때 `WEP`가 보이면 "구형 방식"이라고 이해하면 된다.

### RADIUS

`RADIUS`는 중앙 인증 서버를 통해 사용자 인증을 처리하는 방식이다.

즉 공유기 안에 비밀번호 하나만 저장해두는 게 아니라, 별도의 인증 서버가 있어서 "이 사용자가 접속해도 되는가?"를 확인하는 구조다.

이 방식은 보통:

- 회사
- 학교
- 기관

같이 사용자별 계정 관리가 필요한 환경에서 많이 쓴다.

그래서 `RADIUS`는 "모든 사람이 같은 Wi-Fi 비밀번호를 쓰는 방식"이 아니라, 중앙 서버 기반 인증 방식으로 보면 된다.

### Pre-Shared Key

`Pre-Shared Key`는 미리 공유한 공통 비밀번호를 뜻한다.

가장 익숙한 형태의 Wi-Fi 접속 방식이 이 방식이다. 예를 들면 집에서 Wi-Fi 비밀번호 하나를 정해두고, 모든 기기가 그 비밀번호를 입력해서 접속하는 구조다.

즉:

- `Pre-Shared Key`
  모두가 알고 있는 공통 비밀번호 기반
- `RADIUS`
  중앙 인증 서버 기반

라고 구분하면 된다.

## 한 번에 이해하기

`무선 설정 > 보안 설정`은 "이 Wi-Fi에 누가 어떤 방식으로 들어올 수 있는가"를 정하는 메뉴다.

- `인증모드`
  사용자를 어떤 방식으로 확인할지 정한다.
- `WPA / WPA2 Cipher Suite`
  데이터를 어떤 암호화 방식으로 보호할지 정한다.
- `Key 형식`
  비밀번호 또는 키의 형태를 정한다.
- `Pre-Shared Key`
  실제로 사용하는 공통 Wi-Fi 비밀번호다.

즉 이 메뉴는 단순히 비밀번호 하나 넣는 페이지가 아니라, 무선 네트워크의 `인증 방식`과 `암호화 방식`을 정하는 핵심 보안 페이지라고 이해하면 된다.
